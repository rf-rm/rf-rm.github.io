    1. Application Credentials¶

    -Người dùng có thể tạo ra các thông tin xác thực để cho phép các ứng dụng của họ có thể xác thực với keystone. Người dùng có thể có thể ủy nhiệm các quyền con của người dùng trên một project cho các application credential, cung cấp cho ứng dụng quyền để access tới một project. - Với application credential , các ứng dụng có thể xác thực bằng APP ID và screet string không sử dụng user và password để xác thực . - Khởi tạo một credential cho việc monitoring

    2. Curl API¶

    • Sử dụng CURL đẻ lấy token có giới hạn trong phạm vi project , có thể giới hạn trọng domain hoặc không được khoanh vùng ( không có quyền thực thi ) 
    curl -i \
-H "Content-Type: application/json" \
-d '
{ "auth": {
"identity": {
"methods": ["password"],
"password": {
"user": {
"name": "admin",
"domain": { "id": "default" },
"password": "keystone_123@123Aa"
}
}
},
"scope": {
"project": {
"name": "admin",
"domain": { "id": "default" }
}
}
}
}' \
"http://localhost:5000/v3/auth/tokens"  ;  echo
